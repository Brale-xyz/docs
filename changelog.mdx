<Update label="2026-04-04" description="Week of March 31" tags={["New", "Update", "Fix"]}>
  ## Branded automations

  You can now pass a `brand` object when creating an automation to control the beneficiary name on funding instructions. Set `brand.account_id` to your own or a managed account, and that account's business name appears as the beneficiary instead of the default. See [automations](/key-concepts/automations#branded-automations) for details.

  ## Transfer list filters

  The [List Transfers](/api-reference/brale/list-transfers) endpoint now supports `automation_id`, `value_type`, and `transfer_type` query parameters. Combine multiple filters to narrow results — for example, fetch only Solana USDC transfers created by a specific automation. All filters use exact match with AND semantics.

  ## Address list filters

  The [List Addresses](/api-reference/brale/list-addresses) endpoint now supports filtering by `address`, `type`, and `transfer_type`. Use the `address` parameter to resolve an onchain address to its Brale `address_id` without fetching the full list.

  ## Turnkey Wallet integration guide

  A new [Turnkey integration guide](/guides/integrations/turnkey) walks through connecting Turnkey-managed wallets with Brale for stablecoin onramps, offramps, and onchain transfers.

  ## Pagination parameter update

  The `page[next]` pagination parameter on the List Transfers endpoint has been renamed to `page[after]`. Update your integration accordingly — the value is still taken from the `pagination.next` field in the response. See [transfers](/key-concepts/transfers) for usage examples.
</Update>

<Update label="2026-03-20" description="Week of March 17" tags={["New", "Update"]}>
  ## Tron support

  Brale now supports **Tron** as a transfer type. You can mint, burn, and transfer stablecoins on Tron mainnet and Tron testnet. Use `tron` and `tron_testnet` as `transfer_type` values in the API. See [transfer types](/coverage/transfer-types) for the full list of supported chains.

  ## USDT and RLUSD now available

  Two new third-party stablecoins are available as value types:

  - **USDT** (Tether USD) — issued by Tether
  - **RLUSD** (Ripple USD) — issued by Ripple

  You can now use these alongside Brale-issued stablecoins in transfers. See [value types](/coverage/value-types) for the complete registry.

  ## Transfer timing reference

  A new [transfer timing](/documentation/platform/transfer-timing) page documents cut-off times, settlement windows, and finality expectations for every supported transfer type — including fiat rails (wire, ACH, RTP) and onchain networks grouped by speed tier. Use it to set polling intervals and plan around settlement times.

  ## 20+ supported blockchains

  The platform now supports over 20 blockchain networks for stablecoin issuance and transfers, up from 14. See the full list in [transfer types](/coverage/transfer-types).
</Update>
