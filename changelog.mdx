<Update label="2026-04-18" description="Week of April 14" tags={["New", "Update"]}>
  ## PYUSD support

  **PYUSD** (PayPal USD) is now available as a value type on Ethereum. You can mint, burn, and transfer PYUSD the same way you work with USDC, USDT, and RLUSD. See [value types](/coverage/value-types) and [supported chains by stablecoin](/coverage/transfer-types#supported-transfer_type-values-by-value_type) for details.

  ## Payment details on transfer responses

  Inbound fiat-funded transfers now include `source.payment_details` in the response. This object surfaces metadata from the underlying payment — sender name, bank name, routing number, payment reference, and IMAD — so you can reconcile inbound wires and ACH debits without leaving the API. See [transfers](/key-concepts/transfers#sourcepayment_details) for field descriptions.

  ## Wire memo on outbound transfers

  You can now pass `destination.wire_memo` when creating an outbound wire transfer to include a memo or payment reference for the receiving bank. See the [offramp guide](/guides/stablecoin-to-fiat-offramp) for usage.

  ## Branded automations

  Automations now accept an optional `brand` object so you can control which business name appears as the beneficiary on funding instructions. Pass `brand.account_id` to use your own or a managed account's name instead of the default. See [automations](/key-concepts/automations#branded-automations) for details.

  ## Transfer list filters

  The [List Transfers](/api-reference/brale/list-transfers) endpoint now supports query-parameter filters for `status`, `value_type`, `transfer_type`, `source_address_id`, `destination_address_id`, and `transaction_id`. Combine filters to narrow results — all apply with AND semantics.

  ## Address list filters

  The [List Addresses](/api-reference/brale/list-addresses) endpoint now supports filtering by `address`, `type`, and `transfer_type`. Use the `address` filter to resolve an onchain address to its Brale `address_id`.

  ## Pagination fix

  The transfers pagination parameter has been corrected from `page[next]` to `page[after]`. If you were using `page[next]`, update your integration to use `page[after]` instead.
</Update>

<Update label="2026-03-20" description="Week of March 17" tags={["New", "Update"]}>
  ## Tron support

  Brale now supports **Tron** as a transfer type. You can mint, burn, and transfer stablecoins on Tron mainnet and Tron testnet. Use `tron` and `tron_testnet` as `transfer_type` values in the API. See [transfer types](/coverage/transfer-types) for the full list of supported chains.

  ## USDT and RLUSD now available

  Two new third-party stablecoins are available as value types:

  - **USDT** (Tether USD) — issued by Tether
  - **RLUSD** (Ripple USD) — issued by Ripple

  You can now use these alongside Brale-issued stablecoins in transfers. See [value types](/coverage/value-types) for the complete registry.

  ## Transfer timing reference

  A new [transfer timing](/documentation/platform/transfer-timing) page documents cut-off times, settlement windows, and finality expectations for every supported transfer type — including fiat rails (wire, ACH, RTP) and onchain networks grouped by speed tier. Use it to set polling intervals and plan around settlement times.

  ## 20+ supported blockchains

  The platform now supports over 20 blockchain networks for stablecoin issuance and transfers, up from 14. See the full list in [transfer types](/coverage/transfer-types).
</Update>
