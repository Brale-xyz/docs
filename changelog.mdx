<Update label="2026-05-04" description="Week of April 28" tags={["New", "Update", "Fix"]}>
  ## Webhooks

  Subscribe to platform events and receive HTTP callbacks when transfers, automations, and addresses change state. Includes signed payloads for verification, retry handling, and a managed subscription API. See the [webhooks overview](/webhooks/overview) and [quickstart](/webhooks/quickstart) to get started.

  ## Branded automations

  You can now create branded automations through the API. Use them to issue stablecoins under your own brand on top of Brale's automation flows. See [create automations](/api-reference/brale/create-automations).

  ## Update addresses via API

  A new `PATCH` endpoint lets you archive, unarchive, and update configuration on existing addresses without recreating them. See [update address](/api-reference/brale/update-address) and [addresses](/key-concepts/addresses).

  ## Turnkey Wallet integration

  A step-by-step guide for pairing Brale-managed addresses with [Turnkey](https://www.turnkey.com/) wallet infrastructure is now available. See the [Turnkey integration guide](/guides/integrations/turnkey).

  ## More chains and value types

  - **Starknet** — `starknet` and `starknet_testnet` are available as `transfer_type` values.
  - **PulseChain** — added as a supported `transfer_type`.
  - **PYUSD on Ethereum** — added to the supported value type registry.

  See [transfer types](/coverage/transfer-types) and [value types](/coverage/value-types).

  ## Richer transfer responses

  Transfer responses now expose more detail for fiat rails:

  - **`source.payment_details`** on inbound transfers, including `trace_number` for ACH.
  - **Outbound wire IMAD** on completed wire transfers for end-to-end reconciliation.
  - **`destination.wire_memo`** for outbound wire transfers.

  See [transfers](/key-concepts/transfers) and the [Get Transfer](/api-reference/brale/get-transfer) reference.

  ## New transfer list filters

  [List Transfers](/api-reference/brale/list-transfers) now supports filtering by `transaction_id`, plus additional documented query parameters for narrowing results. [List Addresses](/api-reference/brale/list-addresses) now accepts an `address` query parameter.

  ## Bug fixes

  - **Pagination on List Transfers** — fixed cursor parameter to use `page[after]` instead of `page[next]`. Update any clients that relied on the previous parameter name.
</Update>

<Update label="2026-03-20" description="Week of March 17" tags={["New", "Update"]}>
  ## Tron support

  Brale now supports **Tron** as a transfer type. You can mint, burn, and transfer stablecoins on Tron mainnet and Tron testnet. Use `tron` and `tron_testnet` as `transfer_type` values in the API. See [transfer types](/coverage/transfer-types) for the full list of supported chains.

  ## USDT and RLUSD now available

  Two new third-party stablecoins are available as value types:

  - **USDT** (Tether USD) — issued by Tether
  - **RLUSD** (Ripple USD) — issued by Ripple

  You can now use these alongside Brale-issued stablecoins in transfers. See [value types](/coverage/value-types) for the complete registry.

  ## Transfer timing reference

  A new [transfer timing](/documentation/platform/transfer-timing) page documents cut-off times, settlement windows, and finality expectations for every supported transfer type — including fiat rails (wire, ACH, RTP) and onchain networks grouped by speed tier. Use it to set polling intervals and plan around settlement times.

  ## 20+ supported blockchains

  The platform now supports over 20 blockchain networks for stablecoin issuance and transfers, up from 14. See the full list in [transfer types](/coverage/transfer-types).
</Update>
